# 8

三数之和

题目

# 解题思路

首先想到维护一个map或hash,键是两数的和,值是这两个数分别是什么数.

但受限于题目所述,三元组不能重复,所以需要考虑如何在map中不插入相同的元素,那么这个复杂度就变成了O(n2logn)n^{2}logn)n2logn)

考虑优化,我是没想到,然后搜了下题解,发现都是双指针,往后面搜了下,还是出现了map的解法,但依他所言,map的复杂度也是可以过的

而他处理不重复的方式是这样的:

首先一个map映射全部的数据(可以用hash).
然后另一个map记录结果,最后转成int数组.
去重的方法:

  • 对当前三个数进行排序
  • 如果值全相同,map会自动去重
  • 考虑一个问题,如果c是a或b的情况,那么我们可以在加入前先将a,b从映射map中移出去,然后如果a,b的个数不为0,则允许加入.

然后这种方式被无情的挂在了311/313,但是那个用go + map搞定的和它的结构差不多,所以问题可能出在c++的set比go的差...?还是说必须要手动对三个元素进行排序?还是暂时弃了这种方式吧.

其实这种方法还是可以优化的,如考虑将三个值进行hash,但是需要设计hash算法.

hash_map+set TLE代码 
#include<bits/stdc++.h>
#include<unordered_map>
#include<unordered_set>
using namespace std;

unordered_map<int,int> mv;
set<vector<int>> ans;
set<vector<int>>::iterator it;
vector<vector<int>> answer;
vector<vector<int>> threeSum(vector<int>& nums) {
    if(nums.size()<3) return answer;
    for(int i=0;i<nums.size();++i){
        mv[nums[i]]++;
    }
    for(int i=0;i<nums.size()-1;++i){
        for(int j=i+1;j<nums.size();++j){
            int c = -(nums[i]+nums[j]);
            mv[nums[i]]--;
            mv[nums[j]]--;
            if(mv[c]>0){
                vector<int> vp;
                vp.push_back(nums[i]);
                vp.push_back(nums[j]);
                vp.push_back(c);
                sort(vp.begin(),vp.end());
                ans.insert(vp);
            }
            mv[nums[i]]++;
            mv[nums[j]]++;
        }
    }

    for(it=ans.begin();it!=ans.end();it++){
        answer.push_back(*it);
    }
    return answer;
}


int main(){

    return 0;
}

双指针法.

对于2sum问题而言,常见的方式是一个哈希表映射所有的值,然后O(n)遍历一遍check是否哈希表内有该值,如果有的话,就是答案.

但是还有一种解法是双指针法,为什么双指针法会成立呢?考虑到只需要找到答案,那么数组的顺序完全没有意义

  • 将数组排序
  • 两个指针分别开始于两端,逐渐向中间走,如果相加等于目标值,则成立
  • 放入集合,为保证结果唯一,跳过所有相同的元素.

那么对于3sum问题如何使用双指针法呢?
O(n)遍历所有的元素c,双指针法查找 -c,为O(n^2)

# AC代码


#include<bits/stdc++.h>
#include<unordered_map>
#include<unordered_set>
using namespace std;

vector<vector<int>> threeSum(vector<int>& nums) {
    vector<vector<int>> res;
    sort(nums.begin(),nums.end());
    for(int i = 0; i + 2 < nums.size() ; ++i)       //注意如果是i < nums.size()-2,数组为空时会发生下溢
    {
        if(nums[i] > 0) break;      //后面的数字均大于0，故不存在符合条件的结果
        if(i > 0 && nums[i] == nums[i-1])   continue;       //以nums[i-1]开头的3个数的组合已经判断过

        int target = - nums[i];
        int l = i + 1, r = nums.size() - 1;
        while(l < r)
        {
            int sum = nums[l] + nums[r];
            if(sum == target)
            {
                res.push_back({nums[i],nums[l],nums[r]});
                l++;
                r--;
                while(l < r && nums[l] == nums[l-1])    l++;
                while(l < r && nums[r] == nums[r+1])    r--;
            }
            if(sum < target)  l++;
            if(sum > target)  r--;
        }
    }
    return res;
}

int main(){

    return 0;
}
