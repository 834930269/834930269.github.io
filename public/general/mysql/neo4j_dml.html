-->
-[role]->

-[role:ACTED_IN]->
-[role:ACTED_IN {roles: ["Neo"]}] ->

# 匹配节点(Match)

一个MATCH语句将寻找所有满足匹配条件的路径,并且返回每一行.

查找所有带有Movie标签的数据

MATCH (m:Movie)
RETURN m
1
2

带属性匹配(表中必须存在)

MATCH (p:Person { name: "Keanu Reeves"})
RETURN p
1
2

你必须提供足够确定节点的信息,并非必须提供全部的属性,某些类似于关系数据库中的主键可以作为查找目标

当然,我们也可以找到更多有趣的连接(路径),如Tom Hanks表演的电影以及他饰演的角色.

MATCH (p:Person {name:"Tom Hanks"}) - [r:ACTED_IN]->(m:Movie)
RETURN m.title,r.roles
1
2

它会返回一个表.

# 查询基础上添加新的结构

我们知道,查询会返回一个路径结构,

比如,我们可以先匹配上一个存在的节点,然后基于这个节点新建一个新的节点和连接.

比如,TomHanks在新电影Cloud Atlas中饰演Zachry

MATCH (p:Person { name:"Tom Hanks" })
CREATE (m:Movie { title:"Cloud Atlas",released:2012 })
CREATE (p)-[r:ACTED_IN { roles: ['Zachry']}]->(m)
RETURN p,r,m
1
2
3
4

无论我们是CREATE还是MATCH得到的节点,我们都可以对其进行操作

一般的,当我们在关系数据库中添加/更新数据是,我们一般会先检查是否存在,如果存在,则直接合并到存在的上面.

MERGE (m:Movie { title: "Cloud Atlas"}) ON CREATE SET m.released=2013 RETURN m
1
2

上述语句开头谓词是MERGE,它会使得当m表创建的时候才会更改released为2013,但因为数据库中存在title为Cloud Atlas的Movie,所以找到了,所以不更改.
除非这个Movie不存在才会为2013.

所以请在使用MERGE之前确认你不想创建一个有信息重复的节点.但是在一个大型的图数据库中,必然会存在一些重复信息的节点,所以对于某些节点我们可能会有一些完整性约束或索引,这个我们将在后面进行讨论.

总而言之,如果您想创建一个无信息重复的节点,请使用带有MERGE的CREATE

MERGE也可以用在关系上(保证只联系一次).

MATCH (m:Movie { title: "Cloud Atlas"})
MATCH (p:Person {name: "Tom Hanks"})
MERGE (p)->[r:ACTED_IN]->(m)
ON CREATE SET r.roles = ['Zachry']
RETURN p,r,m
1
2
3
4
5

关系的方向可以是任意的,MERGE将检查关系,并且创建没有重复的关系.

CREATE (y:Year {year:2014})
MERGE (y)<-[:IN_YEAR]-(m10:Month {month:10})
MERGE (y)<-[:IN_YEAR]-(m11:Month {month:11})
RETURN y,m10,m11
1
2
3
4

其中任意一条没有的节点或者关系都会被创建,有的话就不创建.

# 过滤结果

我们可以使用MATCH谓词得到所有我们想要的结果,现在我们想要对结果进行过滤,来得到我们感兴趣的结果.
所有的过滤条件将使用WHERE子句,这个子句将用于任意个数的表达式/验证. 如: AND OR XOR(异或) NOT

一个简单的例子:

MATCH (m:Movie)
WHERE m.title = "The Matrix"
RETURN m

上述简单的过滤语句等价于:

MATCH (m:Movie { title: "The Matrix" })

# 返回结果(如表格形式)

在之前,我们一直返回的都是节点/关系或者路径,那么我们如何返回任意一个数字或者表达式呢?那么在Cypher中表达式又是什么呢?
一个最简单的可迭代的表达式.
如

  • numbers
  • strings
  • arrays(for example: [1,2,3])
  • maps(for example: {name:"Tom Hanks",born:1964,movie:["Forrest Gump",..],count:13})

在Cypher中,关系/节点/路径的私有变量都可以用dot运算符来取得.如: n.name 数组类的也可以通过...运算符来切割,或者按下标取值:
names[0] OR movies[1...-1]每个函数都有自然属性.如 length(array),toInteger("12"),substring("2014-07-01",0,4)或者 coalesce(p.nickname,"n/a")

当然,全部函数应该在手册中寻找

若想通过表达式返回表格(Mysql那种)以及更改名字为可理解的,则需要使用expression AS alias.

MATCH (p:Person)
RETURN p,p.name AS name,toUpper(p.name),coalesce(p.nickname,"n/a") AS nickname,{name:p.name,label:head(labels(p))} AS person

# count

返回数据库中人标签节点的总数

MATCH (:Person)
RETURN count(*) AS people
1
2

注意,空节点会自动跳过,不管是否有标签.如果想要统计不同姓名的人的个数:

count(DISTINCT role).
1

在count聚合函数中,Cypher将使用所有的非聚合列作为分组的键. 如查找每个导演和每个演员合作的电影的个数

MATCH (actor:Person)-[:ACTED_IN]->(movie:Movie)<-[:DIRECTED]-(director:Person)
RETURN actor, director, count(*) AS collaborations
1

# 排序和分页

当我们使用count(*)以后可以很自然地进行排序,排序将使用ORDER BY expression [ASC|DESC]子句.

其实我们可以依照多字段排序或者排序后返回其他的字段,如我们返回person.name,但我们可以按照person.age进行排序.我们不能以不能返回的值来排序.

分页将使用 SKIP {offset} and LIMIT {count}子句.

# 返回Top-n实体

返回演电影最多的10个演员

MATCH (a:Person)-[:ACTED_IN]->(m:Movie)
RETURN a,count(*) AS appearances
ORDER BY appearances DESC LIMIT 10;
1
2

# 收集聚合函数(collect())

collect（）非常适合于检索典型的父子结构，其中每行返回一个核心实体（parent、root或head），并在collect（）创建的相关列表中包含其所有依赖信息。这意味着不需要为每个子行重复父信息，也不需要运行n+1语句来分别检索父行及其子行。

上面直接翻译的...举个栗子

MATCH (m:Movie)<-[:ACTED_IN]-(a:Person)
RETURN m.title AS movie, count(*) AS actors
1
2
3

上面将根据聚合函数count(*)直接返回以title作为键的所有分组的个数.
如果我们想知道都有哪些演员呢?
为了避免对于每一行都在进行一次查找,提供了collect函数.

MATCH (m:Movie)<-[:ACTED_IN]-(a:Person)
RETURN m.title AS movie, collect(a.name) AS cast, count(*) AS actors
1
2

这样就可以了. 翻译: 查询所有电影,以及该电影的演员个数与所有演员.

# WITH

在Cypher中，可以将语句的片段链接在一起。每个片段对前一个片段的输出起作用，其结果可以反馈到下一个片段中。只有在WITH子句中声明的列才可用于后续查询部分。

WITH子句用于组合各个部分，并声明哪些数据从一个部分流向另一个部分。WITH与RETURN子句类似。区别在于WITH子句没有完成查询，但为下一部分准备输入。

表达式、聚合、排序和分页与RETURN子句中的用法相同。唯一的区别就是所有的列必须是不同的别名。

MATCH (person:Person)-[:ACTED_IN]->(m:Movie)
WITH person, count(*) AS appearances, collect(m.title) AS movies
WHERE appearances > 1
RETURN person.name, appearances, movies
1
2
3
4
+-------------------------------------------------------------+
| person.name | appearances | movies |
+-------------------------------------------------------------+
| "Tom Hanks" | 2 | ["Cloud Atlas", "Forrest Gump"] |
+-------------------------------------------------------------+
1 row

# 索引INDEX

添加索引可以更快的找到图的起点

但需要注意的是,如果对已经有数据的数据库添加索引,需要添加一段时间.

比如,我们想通过演员的名字来查找演员:

CREATE INDEX ON :Actor(name)
1

在大多数情况下,查询时不需要指定索引列,只要你指定了属性参数,他就会自动开始索引.

MATCH (actor:Actor {name: "Tom Hanks"})
RETURN actor;
1
2

复合索引

我们也可以根据多字段添加索引,如我们可以添加演员的name和born作为索引属性,但值得注意的是,有的演员没有born信息,对于复合索引而言,必须两个信息都存在才会被加入到索引中.

CREATE INDEX ON :Actor(name,born)
1

我们也可以通过数据库的内置命令查找我们之前定义的所有索引.

CALL db.indexes
YIELD description,tokenNames,properties,type;
1
2
3

这个CALL后面的YIELD会出错,但是只执行CALL db.indexes没问题.

2、然后我们将文件加载进来

LOAD CSV WITH HEADERS FROM "file:///persons.csv" AS csvLine
CREATE (p:Person {id: toInteger(csvLine.id), name: csvLine.name})
	
LOAD CSV WITH HEADERS FROM "file:///movies.csv" AS csvLine
MERGE (country:Country {name: csvLine.country})
CREATE (movie:Movie {id: toInteger(csvLine.id), title: csvLine.title, year:toInteger(csvLine.year)})
CREATE (movie)-[:MADE_IN]->(country)

USING PERIODIC COMMIT 500  //大文件进行分批导入，默认1000行提交一次，这里设置为500行提交一次
LOAD CSV WITH HEADERS FROM "file:///roles.csv" AS csvLine
MATCH (person:Person {id: toInteger(csvLine.personId)}),(movie:Movie {id: toInteger(csvLine.movieId)})
CREATE (person)-[:PLAYED {role: csvLine.role}]->(movie)
