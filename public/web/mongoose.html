# 索引

在关系数据库中,索引既包含实体完整性约束,也包含索引映射

如UNIQUE,NOT NULL,PRIMARY KEY等都是索引.

设置索引会导致表的增加速度变慢,(因为要进行额外的索引映射操作)

# 数据校验

即用户自定义完整性约束,如最大值约束,最小值等等

required : 表示这个数据必须传入 
max: 用于 Number 类型数据，最大值 
min: 用于 Number 类型数据，最小值 
enum:枚举类型，要求数据必须满足枚举值 enum: ['0', '1', '2'] 
match:增加的数据必须符合 
match（正则）的规则 
maxlength：最大值 
minlength：最小值
unique:true
type: Schema.Types.ObjectId: (外键)

# Mongoose自定义的验证器

在缺省情况下创建的索引均不是唯一索引,下面创建唯一索引

var UserSchema = new mongoose.Schema({
	name:{
		type:String,
		require:true,
	},
	age:{
		type:Number,
		require:true,
		max:120,
		min:0
	},
	status:{
		type:String,
		enum:['0','1','2'],
	},
	phone:{
		type:Number,
		match:/^\d{11}$/,
	},
	desc:{
		type:String,
		//自定义验证器,通过验证返回true,不通过返回false
		validate:function(desc){
			return desc.length >= 10;	
		}
	}

});

# 表关联查询,聚合管道

如果console.log出来的存在Object,可以使用JSON.stringfy()转换一下

这个语法需要直接把聚合管道写上去

如查询order表与order_item

//OrderModel写在其他文件里,暴露出来的

OrderModel.aggregate([
	{
		$lookup:
			{
				from: "order_item",
				localField: "order_id",
				foreignField: "order_id",
				as: "items"
			}
	}
],function(err,docs){
	if(err){
		console.log(err);
		return;
	}

	console.log(JSON.stringify(docs));
})
