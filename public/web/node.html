# Node.js

摘自官档,慢慢补充

Node.js的由来:

以前谷歌为了让前端更好/快的运行,开发了V8与IE竞争,最后IE失败了,然后V8开源了,总而言之,V8就类似于一个Javascript的虚拟机.

Ryan Dahl想要开发一个高性能的Web引擎,但他不想自己开发,所以对V8改了一下搞出了个Node.js,为什么他要用Javascript来做这件事呢?因为很多人用静态语言最后都会习惯使用同步来写代码.但是javascript因为其特性所以处处都是异步.

# npm是啥玩意

npm是Node.js的包管理工具(package manager).
大家把自己开发的模块打包后放到npm官网上,就不需要一个一个的去找了.
npm也可以自定义依赖,而不需要一个个包去下载.
npm一般自带与node.js

# CommonJS规范

Node模块加载机制被称为CommonJS规范,这个规范下,每个.js都是一个模块,他们内部的变量名和函数名互不冲突.

# CommonJS实现原理

当我们编写JavaScript代码的时候,我们可以声明全局变量


var s = 'global';
1
2
3

在浏览器中,使用全局变量会造成冲突,如b.js有变量s,a.js也有变量s.

Js本身并不支持模块这个功能,但因为JS是一种函数式编程语言,它支持闭包,所以我们可以把一段段的JS代码包装起来,这段代码的全局变量就变成了局部变量.

例子

hello.js

var s = 'Hello';
var name = 'world';

console.log(s + ' ' + name + '!');

(function(){
	//读取的hello.js的代码
	var s = 'Hello';
	var name = 'world';
	
	console.log(s + ' ' + name + '!');
	//hello.js代码结束
})();

		console.log('Hello, '+name+'!');
	}
	module.exports = greet;

	return module.export;

var exported = load(module);

# EventEmitter

const EventEmitter = require('events');
1

Node的事件模块中，目前只包含一个类:EventEmitter。这个类在Node的内置模块中被大量使用，在Node中，所有能触发事件的对象都是 EventEmitter 类的实例。要使用 EventEmitter，首先必须要继承它。

理解为: 一个对象会有一些事件,而这些事件触发后需要进行一些操作,此时我们可以用 .on来监听事件

const EventEmitter = require('events');
//所有的构造函数都必须继承自EventEmitter类;
class MusicPlayer extends EventEmitter{};
//再通过这个构造函数来创建触发事件的对象
let musicPlayer = new MusicPlayer();
let AudioDevice = {
  play:function(track){
    //
  },
  stop:function(){
    //
  }
}
//监听事件
musicPlayer.on('play',function(track){
  this.playing = true;
  AudioDevice.play(track);
})

.on('事件名‘，function（）{}） ---相当于 bind，但是不会触发

# unhandleRejection事件

如果在事件循环的一次轮询中，一个 Promise 被拒绝，并且此 Promise 没有绑定错误处理器， 'unhandledRejection 事件会被触发。 当使用 Promise 进行编程时，异常会以被拒绝的 promise 的形式封装。 拒绝可以被 promise.catch() 捕获并处理，并且在 Promise 链中传播。'unhandledRejection 事件在探测和跟踪 promise 被拒绝，并且拒绝未被处理的场景中是很有用的。

。。。。。大概意思是,Promise出现错误不处理,直接丢给这个事件
