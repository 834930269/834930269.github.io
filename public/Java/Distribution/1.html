# 为什么要开分布式?

       想要开发一个接口测试的App,以后做项目大概都是前后端分离了.如果每次都使用Postman感觉有点繁琐.

# 什么是分布式系统?

  • 很多计算机组成一个整体,一致对外处理同一请求
  • 简而言之,类似于CDN,只不过逻辑层面上是一体的
  • 内部的每台计算机都可以互相通讯
  • 一次请求会经过多台计算机
  • 分布式 < 集群

       那么为什么要这样搞?

`点开查看答案`

有些时候一个电脑会无法处理同一时刻到来的请求. 如该请求有几个需求:

  • 需要处理图片

  • 需要调用人工智障

  • ...
    一大堆的需求同时出现在一台电脑上是肯定无法完成任务的. 所以可以考虑这种拆分方法:

  • 将图片处理分配给一台电脑

  • 将人工智障分配给一台电脑

  • ...

√

# zookeeper特性

  • 一致性: 数据一致性,数据按照顺序分批入库
  • 原子性: 事务要么成功要么失败,不会局部化.
  • 单一视图: 任何连接到集群上的机器所读取到的状态都是一样的
  • 可靠性：每次对zk的操作状态都会保存在服务器,可以回滚
  • 实时性: 客户端可以读取到zk服务端的最新数据
